API는 키보드 같은 것이다.

키보드로 정보를 입력하여 모니터에 글자가 출력되거나, 여러 기능을 실행시킬 수 있다.

이렇게 컴퓨터와 소통하는 것처럼. 프로그램이 서로 소통하는 방법이 API 이다.
=> 코드들끼리 서로 소통하기 위해 만들어진 것이 API이다.

API는 서버에서 만든 것이다. 그리고 그 API를 특정 앱에 적용시키는 것.
여러 API를 적용시켜서 키보드 라는 것을 만들었다고 가정하자.
"A라는" 앱에서 z버튼은 화면켜기, X버튼은 실행취소 등 여러 API가 적용됨.

faceBook API, Graph API 등등 ~~~ API 들의 최종 목적은 기능이 다를지언정.
프로그램들이 서로 소통할 수 있도록 도와주는 것이다. 
=> 모든 API는 프로그램들끼리 소통을 도와준다.

Web API는 브라우저를 위해 만든 API를 뜻한다.
=> 구글 크롬, 파이어폭스 등 브라우저에서 공통적으로 실행되는 기능 제공.

API는 사용 제약이 있거나, 금액을 지불해야할 때도 있다.
=> 설정이 가능하도록 만들면 됨.

API는 내가 원하는 정보, 나의 주변 정보 등 일일히 기입하지 않고도 사용가능하다.
다만, 그 정보들은 어떻게 어디서 제공되는지 하나하나 알 수 없지만, 버튼 하나로 사용 가능하다.
