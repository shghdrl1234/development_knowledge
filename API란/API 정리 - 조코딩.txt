프론트엔드와 백엔드를 연결하는 API 개념과 활용
프론트엔드는 화면을 의미하고
백엔드는 화면에 보이는 정보를 처리하는 곳이다.

프론트엔드가 백엔드에 요청을 할 때 특정 규칙에 맞게 요청 해야한다.
이 특정 규칙을 제공하는 것을 API라고 한다.

open API는 백엔드를 만들어놓고 주소와 사용 규칙을 공개한 것을 의미한다.
이 백엔드의 주소와 사용규칙을 알면, 프론트엔드만 만들어도 얼마든지 요청해서
데이터를 가져와 사용할 수 있다.

공개된 데이터만 사용하고 프론트엔드만 만들어 쓰는 것을 sercerLess라고 부르기도 한다.

API에는 다양한 것들이 있는데 지도, 결제, 위치, 날씨, AI 등이 있다.

규칙에 맞게 데이터만 전송하면 사용가능하다.

API의 서버 주소로 데이터를 요청하면, API 에서는 원하는 정보를 준다.
이 때, API 가이드에 따라 요청과 응답에 대한 형식을 맞춰야한다.
